Declarative Policy Profile

Select a predefined profile or reload the rules file. These profiles define how strictly your node filters transaction spam (e.g. runes, ordinals, OP_RETURN outputs).

Maximalist

Ultra strict

Zero tolerance for data carrier spam. OP_RETURN is blocked entirely, and all inscriptions/token protocols are actively filtered from the mempool.

BIP-110 Strict

Consensus cap

Enforces standard BIP-110 consensus bounds locally, capping pushdata elements and transaction output script sizes.

Monetary Only

Financial focus

Relays and blocks only financial Bitcoin transactions. Blocks OP_RETURN outputs and inscription/token structures aggressively.

Default Knots

Standard

Uses standard Bitcoin Knots relay and policy configuration rules. No additional filtering.

Custom

Advanced

Use custom_rules from policy.toml. Edit in Configuration → Policy, then Apply Policy on the dashboard.
